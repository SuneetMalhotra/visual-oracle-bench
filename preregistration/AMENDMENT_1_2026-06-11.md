# OSF Amendment 1 — 2026-06-11

**Registration:** Visual Oracle Bench (OSF DOI 10.17605/OSF.IO/NKD6J, registered 2026-06-06)
**Type:** Dated supplemental amendment record. This is NOT a re-registration; the pre-registered Phase 2 design and analysis plan are unchanged.

---

During Phase 1 analysis we discovered that the original judge wrappers
(commit `eaf1e4d`, 2026-06-06) silently defaulted to `verdict=fail` with
`malformed=false` when providers returned rate-limit or auth-error
responses. We patched the wrappers (commits `9352483`, `da1f70a`),
re-flagged 617 affected rows as malformed in the analyzer
(`analysis/analyze_judgments.py::_flag_silent_fabrications()`), ran a
Codex-only top-up on the 200 control pairs (commit `0745d7b`), and
report Phase 1 results on the integrity-audited subset. The
pre-registered Phase 2 design and analysis plan are unchanged; this
amendment documents the Phase 1 correction protocol.

This amendment additionally documents two post-registration Phase 1
decisions, both made in response to independent adversarial review of
the v1 manuscript draft (2026-06-10):

1. **Control-pair extension.** 200 identical-image control pairs
   (baseline = candidate; expected verdict `pass`) were added to the
   Phase 1 corpus as manifest v2 (`data/images/_pairs_manifest_v2.json`,
   600 pairs total), so that specificity, precision, F1, and MCC are
   defined. Generation script: `scripts/add_true_negatives.py`.

2. **Llama exclusion from comparative analysis.** Llama 3.2 Vision 11B
   rejects multi-image requests; the side-by-side composite workaround
   changes the input modality relative to the other judges, confounding
   capability with format. Llama is therefore excluded from all headline
   comparative metrics in Phase 1 reporting and documented as an
   integration-failure finding in the manuscript's Appendix A. The
   Phase 2 judge roster in the registration is unaffected.

The contaminated original parquets are retained unmodified in the
released artifact; the exclusion is recomputed at every analyzer run and
is therefore independently auditable.
